# Omnilytix: Autonomous AI Data Analyst for SQL & MongoDB

Omnilytix is a comprehensive, AI-powered Data Analyst application. Built on top of **LangChain**, **Streamlit**, and the **Groq API**, Omnilytix transforms natural language questions into database queries, parses the results, and automatically generates interactive insights, structured data tables, and advanced visualizations.

## 🚀 Key Capabilities

### 1. Advanced Data Visualization (PowerBI Style)
Omnilytix doesn't just return text—it is a fully capable visual data analyst. Using a secure tool-calling architecture, the AI intercepts raw JSON data and natively renders **10 different chart types** without ever executing unsafe Python code on your machine:
- **Plotly Express (Interactive)**: Bar/Column, Line, Scatter, Pie, Donut, Area, Funnel, Treemap.
  - *Features interactive tooltips, zooming, panning, and native PNG exports.*
- **Matplotlib & Seaborn (Statistical)**: Heatmap (Z-index or Correlation matrices), Box plots, Histograms.

### 2. Multi-Database Support
Omnilytix seamlessly connects to both relational and NoSQL databases:
- **SQL Databases**: PostgreSQL, MySQL, SQL Server. Uses Langchain's `SQLDatabaseToolkit` optimized for Qwen/Llama models.
- **MongoDB**: Uses a highly customized, read-only toolkit capable of executing collections listings, counting, sampling, and securely validated `$aggregate` pipelines.

### 3. Secure by Design
- **Read-Only Enforcement**: Strict system prompts prevent the AI from generating DML statements (INSERT, UPDATE, DELETE, DROP).
- **Safe Aggregations**: MongoDB pipelines are validated against an explicit allowlist of safe read-only operations (`$match`, `$group`, `$sort`, etc.).
- **No Arbitrary Code**: Visualizations are generated by parsing strict JSON contracts (`data_json`, `chart_type`, `x_key`, etc.) directly into Pandas DataFrames, preventing any remote code execution risks.

### 4. Automated Data Profiling (EDA)
Behind the scenes, Omnilytix runs a hyper-optimized **Pandas Background Interceptor** on every data retrieval tool (`sql_db_query`, `find_mongo_documents`, etc.).
- **Zero Token Overhead**: It mathematically checks the dataset locally for Missing Values (>5%) and Extreme Outliers (using IQR boundaries) without feeding raw datasets to the LLM.
- **Proactive AI**: Finds are condensed into a lightweight tag (`[EDA Profile: ...]`) and appended to the tool execution, prompting the AI to proactively warn you in its response (e.g., *"Here is your data, but please note that the 'Revenue' column has 12 outliers."*).

### 5. Rich Chat Interface (Streamlit)
The `app.py` UI is heavily customized for data analysis:
- **Query Transparency**: Every SQL or Mongo query executed by the AI is displayed in a collapsible "Executed Query" expander so you can audit exactly how the data was retrieved.
- **Native DataFrames**: Arrays of records are automatically detected and rendered as interactive Streamlit tables.
- **Rich Media Extraction**: The UI automatically parses image URLs and markdown links from the database and renders them as proper UI components.
- **Zero `.env` Hassle**: All database URIs, configurations, and API keys are securely entered directly into the Streamlit sidebar.

## 🛠️ Installation & Setup

### Prerequisites
- Python 3.10+
- A Groq API Key

### Dependencies
Omnilytix relies on a robust data stack:
```bash
pip install streamlit pandas plotly matplotlib seaborn requests python-dotenv
pip install langchain langchain-community langchain-groq
pip install sqlalchemy pymysql psycopg2-binary pyodbc pymongo
```
*(Note: If you are connecting to SQL Server, ensure the `ODBC Driver 18 for SQL Server` is installed on your OS).*

## 🖥️ Usage

### 1. Streamlit Web UI (Recommended)
Launch the interactive web interface:
```bash
streamlit run app.py
```
1. Open the sidebar on the left.
2. Enter your **Groq API key** (this is required to power the AI).
3. Select your **Connection type** (`SQL database` or `MongoDB`).
4. Enter your connection details:
   - **For MongoDB**: Since the app is deployed, you must use a cloud database provider (like MongoDB Atlas). Paste your **Atlas Connection String** (e.g., `mongodb+srv://user:pass@cluster0.mongodb.net/`) into the `MongoDB URI` box and specify the `MongoDB database` name.
   - **For SQL**: Select your `Database type` (PostgreSQL, MySQL, or SQL Server). Since the app is deployed, you must use a cloud database provider (like Render or Supabase). Paste your **External Database URL** (e.g., `postgres://user:pass@host.render.com/db`) directly into the **"Custom Connection URI"** box, OR manually enter the Cloud Host, Port, Username, and Password.
5. Click the **Connect** button.
6. Once connected, use the chat box to ask analytical questions in plain English!
   - *"Show me the top 5 highest paying customers as a bar chart."*
   - *"Create a donut chart of sales broken down by region."*
   - *"Generate a heatmap of our user behavioral correlations."*

## 🏗️ Architecture

- **`app.py`**: The Streamlit frontend. Handles session state, dynamic chart rendering (`st.plotly_chart`, `st.pyplot`), dataframe parsing, and markdown extraction.
- **`sql_agent.py`**: The core LLM orchestration logic. Defines the strict `SYSTEM_PROMPT_TEMPLATE` that forces natural language conversation (no raw JSON dumping) and directs the AI to query the DB *before* invoking charting tools.
- **`tools.py`**: Contains the customized MongoDB toolkit and the highly structured `generate_chart` tool schema.
- **`database.py`**: Manages the SQLAlchemy engines and PyMongo client connections.
